## 信号是什么

观潮的"信号"不是单一技术指标，是一个**8维综合评分**经过**体制调整**后的最终判断。

每次调用 `analyze_stock` 时，系统会：

1. **8维技术评分**（每个维度独立打分，加权汇总）：
   - MA（均线趋势）— 多头/空头排列、站上/跌破MA5/MA20
   - RSI（超买超卖）— RSI14 位置
   - MACD（趋势动量）— 金叉/死叉、红绿柱放大
   - KDJ（短期超买超卖）— J值位置
   - BOLL（波动区间）— 触及上下轨
   - ATR（波动率）— 近5日涨跌幅
   - OBV（量价关系）— 资金流入/流出斜率
   - 形态识别 — 缩量/放量、换手率异常

2. **体制调整**（regime bias）：
   - 识别大盘处于 bull/mild_bull/sideways/mild_bear/bear/high_volatility
   - 给评分加偏移：bull +15, bear -15 等
   - 结果：`adjusted_score = raw_score + regime_bias`，范围 [-100, +100]

3. **信号生成**：
   - ≥ +50 → 看多
   - +25~+50 → 偏多（bear/mild_bear 下强制中性）
   - -25~+25 → 中性
   - ≤ -25 → 看空

4. **冲突检测**叠加：技术看多但资金流出？个股强但大盘弱？这些矛盾信号单独标注。

所以「信号」= **传统信号组合 + 体制感知 + 冲突检测**的综合判断，不是单一指标。

## 信号回填是什么

每次分析会自动记录一条信号到 SQLite：
```
{symbol, score, direction, price_at_signal, regime, conflicts, timestamp}
```

**回填**是事后验证——5/10/20 个交易日后，拉当时的收盘价，算涨跌幅，判断信号对不对：

- 看多 + 实际涨了 → correct
- 看空 + 实际跌了 → correct
- 其他 → wrong

这就是"回填"——**不是回测**（回测是拿历史数据模拟交易），而是对**实际发出的信号**做事后验证。

## Alpha 怎么来的

坦率说，观潮的 alpha 来源很朴素：

1. **技术面**：标准技术指标（MA/RSI/MACD/KDJ/BOLL/OBV），没有自创指标。alpha 不在单个指标，在于 8 维交叉验证和阈值调参
2. **体制感知**：同样的 +30 分，在牛市里是"顺势"，在熊市里是"逆势"。体制调整让信号有了市场上下文
3. **冲突检测**：散户最容易忽略的矛盾信号——"技术面看多但主力在出货"。这是观潮的核心差异化
4. **行为护栏**：不是 alpha 本身，但防止你在最差的时机行动（追高、连续看空还去看）

**没有机器学习**，没有因子模型，没有高频数据。这是一个规则引擎 + LLM 叙事的投研搭档，不是量化交易系统。
